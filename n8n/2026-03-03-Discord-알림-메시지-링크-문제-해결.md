### Context
GitHub에 새 글이 업로드되면 Discord로 알림 메시지를 전송하는 n8n 워크플로를 구축했습니다. 그러나, Discord 채널에 나타나는 메시지의 하이퍼링크가 파란색으로 활성화되지 않고 일반 텍스트로 표시되는 문제가 있었습니다. 이로 인해 GitHub 문서의 미리보기(Embed) 카드도 나타나지 않았습니다.

### Core
Discord 메시지는 마크다운(Markdown) 문법을 지원하며, 링크 생성 시 `[화면에 보일 텍스트](연결할 URL)` 형식을 따라야 합니다. 문제는 n8n의 Discord 전송 노드 설정에서 URL이 들어가는 괄호 안에 불필요한 대괄호가 포함된 오타였습니다.

오류 구문 예시:
```
[👉 기록된 TIL 바로보기](https://github.com/.../blob/main/[n8n/파일명.md)
```
Discord가 마크다운 문법을 정상적으로 파싱하지 못하도록 했던 원인은 URL 경로에 있는 불필요한 `[ `기호였습니다. 이를 해결하기 위해, Discord 메시지 전송 노드 설정에서 이 기호를 삭제했습니다.

수정 전:
```
.../blob/main/[{{ $json.folder }}/...
```
수정 후:
```
.../blob/main/{{ $json.folder }}/...
```

### Insight
마크다운 문법은 작은 기호나 띄어쓰기 하나에도 민감하게 반응합니다. 눈에 띄는 오류 메시지가 발생하지 않더라도 결과물이 예상과 다르게 나타난다면, 작성한 코드나 문구에 작은 '오타'나 '문법 규칙 위반'이 없는지를 꼼꼼히 확인해야 합니다.
