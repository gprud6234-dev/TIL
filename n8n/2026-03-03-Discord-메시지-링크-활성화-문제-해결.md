### Context
n8n 워크플로를 통해 GitHub에 새 글이 업로드될 때, Discord로 알림 메시지를 전송하도록 설정했습니다. 그러나 Discord 채널에 전송된 메시지의 하이퍼링크가 파란색으로 활성화되지 않고 일반 텍스트로 표시되는 문제가 발생했습니다. 이로 인해 GitHub 문서의 미리보기(Embed) 카드도 나타나지 않았습니다.

### Core
Discord 메시지는 마크다운(Markdown) 문법을 지원하며, 링크 생성 시 `[화면에 보일 텍스트](연결할 URL)` 형식을 엄격히 지켜야 합니다. n8n의 Discord 전송 노드 설정에서 URL이 포함된 소괄호 영역에 불필요한 대괄호 기호가 잘못 포함되었습니다.

* 오류 구문: `[ 👉 기록된 TIL 바로보기](https://github.com/.../blob/main/[n8n/파일명.md)`
  * URL 경로 `main/` 바로 뒤에 `[ 기호가 있어 마크다운 문법이 올바르게 파싱되지 않았습니다.

* 수정 전: `.../blob/main/[{{ $json.folder }}/...`
* 수정 후: `.../blob/main/{{ $json.folder }}/...`

### Insight
마크다운 문법은 작은 기호나 띄어쓰기에 민감하게 반응합니다. 눈에 띄는 오류 메시지가 없더라도 결과가 예상과 다르다면 작성한 코드나 문구에 오타나 문법 규칙 위반이 없는지 점검하는 습관이 필요합니다.
