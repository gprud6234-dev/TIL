### Context
n8n과 ngrok를 함께 사용하여 외부 액세스를 가능하게 하는 워크플로우를 구성하고자 한다. ngrok는 일반적으로 로컬 서버를 인터넷에서 접근 가능하도록 터널링하는 도구이다. 이를 n8n과 함께 사용하면, 외부에서 n8n의 워크플로우를 실시간으로 테스트하거나 외부 API와 연결이 용이해진다.

### Core
n8n을 ngrok와 함께 사용하는 방법은 도커(Docker)를 활용하여 두 서비스를 함께 실행하는 두 가지 방법이 있다.

* 도커 컴포즈(Docker Compose)를 사용하여 둘 다 동시에 실행
* 별도의 도커 컨테이너로 각각 실행

아래는 Docker Compose를 사용하여 n8n과 ngrok를 동시에 실행하는 예시이다.

```yaml
docker-compose.yml

version: '3.1'

services:
  n8n:
    image: n8nio/n8n
    ports:
      - "5678:5678"

  ngrok:
    image: wernight/ngrok
    ports:
      - "4040:4040"
    command: ngrok http n8n:5678
```

### Insight
이 설정을 통해 ngrok는 n8n의 로컬 서버를 외부에서 접근할 수 있도록 터널링한다. Ngrok는 자동으로 HTTPS URL로 변환하여 제공하므로, 이를 통해 보안 설정도 쉽게 반영할 수 있다. 이러한 설정은 실시간 외부 API 연동 테스트 및 개발 환경에서 매우 유용하게 사용될 수 있다.
