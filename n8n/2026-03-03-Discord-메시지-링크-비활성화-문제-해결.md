### Context
n8n 워크플로를 통해 GitHub에 새 글이 업로드되면 Discord로 알림 메시지를 전송하도록 설정했습니다. 그러나 Discord 채널에 전송된 메시지에서 링크가 파란색 하이퍼링크로 활성화되지 않고 일반 텍스트로 노출되며, GitHub 미리보기(Embed) 카드도 생성되지 않았습니다.

### Core
Discord 메시지는 마크다운(Markdown) 문법을 지원합니다. 링크를 생성할 때는 `[화면에 보일 텍스트](연결할 URL)` 형식을 엄격하게 따라야 합니다. n8n의 Discord 전송 노드 설정을 확인한 결과, URL 변수 끝에 닫는 소괄호 `)`가 누락되어 Discord가 이를 링크 문법으로 인식하지 못한 것을 발견했습니다.

**오류 구문**: `.../blob/main/{{ $json.folder }}/{{ $json.filename }}`  
(맨 끝에 `)`가 빠져있어 마크다운 규칙이 깨짐)

**해결 방법**: n8n 워크플로의 Discord 메시지 전송 노드 설정(Content)에서, 링크 주소 맨 끝에 닫는 소괄호 `)`를 추가했습니다.

* **수정 전**: `.../{{ $json.filename }}`
* **수정 후**: `.../{{ $json.filename }})`

### Insight
코드나 마크다운을 작성할 때 괄호의 짝(열고 닫기)을 맞추는 것은 매우 기본적이지만 가장 놓치기 쉬운 부분입니다. 에러 메시지가 명확하게 뜨지 않고 결과물의 형태가 이상하다면, 가장 먼저 '오타'나 '괄호/따옴표 등의 닫힘 상태'를 꼼꼼히 점검하는 습관을 가져야겠습니다.
